### Multi-head Latent Attention (MLA)

MLA 是 DeepSeek 最核心的架构创新。传统 MHA（Multi-Head Attention）在推理时需缓存完整的 KV 矩阵，而 MLA 通过将 Key 和 Value 映射到低维潜在空间，大幅压缩 KV 缓存：

| 指标 | MHA（LLaMA-70B） | MLA（DeepSeek-V3） |

| 每层 KV 缓存大小 | 2 × 128 × 4096 | 2 × 64 × 512 |
| 显存占用（批大小 1） | ~2.1 GB | ~0.13 GB |
| 推理吞吐量 | 基线 | **提升 4.2 倍** |

### MoE 训练优化

DeepSeek-V3 采用 Mixture-of-Experts 架构，总参数量 671B，但每个 token 仅激活 37B 参数。其核心训练技巧包括：

- **负载均衡损失**：通过 auxiliary loss 确保专家间的负载均衡
- **Token 丢弃策略**：当某个专家过载时，丢弃部分 token（而非重新路由），保证计算图稳定
- **FP8 混合精度训练**：首次在千亿级 MoE 模型上验证了 FP8 训练的有效性

```python
# MoE 负载均衡损失简化实现
def load_balancing_loss(gate_logits, expert_indices, num_experts):
    # gate_logits: (batch, seq, num_experts)
    # expert_indices: (batch, seq)
    tokens_per_expert = torch.bincount(
        expert_indices, minlength=num_experts
    ).float()
    fraction_per_expert = tokens_per_expert / tokens_per_expert.sum()
    
    # 理想情况下，每个专家应处理等量 token
    ideal_fraction = 1.0 / num_experts
    loss = (fraction_per_expert - ideal_fraction).pow(2).sum()
    return loss * LOAD_BALANCING_COEFF
```

## LLM 推理优化：Speculative Decoding 实践指南

推测解码（Speculative Decoding）是一种在不影响生成质量的前提下加速 LLM 推理的技术。其核心思路是**用小模型起草、大模型验证**：

```
Step 1: 草稿模型快速生成 K 个候选 token
Step 2: 目标模型并行验证这 K 个 token
Step 3: 接受匹配的 token，丢弃第一个不匹配之后的 token
```

### 工程实践要点

1. **草稿模型选择**：理想情况下，草稿模型的接受率应 > 70%。通常选择参数量为目标模型的 5-10% 且共享词表的模型
2. **K 值调优**：K 值过大浪费草稿计算，过小降低加速效果。实践经验表明 K=5 是多数场景的甜点
3. **采样兼容性**：推测解码与 temperature sampling、top-k/top-p 采样天然兼容

实际部署中，推测解码在 70B 模型上可实现 **1.8-2.5 倍** 的推理加速，且输出分布与标准自回归解码一致。
